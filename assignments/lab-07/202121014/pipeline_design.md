# Multi-Agent Pipeline Design

## Planner

역할:
- 사용자 목표를 작업 가능한 계획으로 분해한다.
- 전체 작업 순서와 제약 조건을 정의한다.
- 이후 에이전트가 사용할 공통 `task_id`를 만든다.

## Researcher

입력:
- 사용자 목표
- 코드베이스 요약
- Planner가 만든 상위 계획

출력:
- 조사 메모
- 관련 파일 목록
- 확인이 필요한 위험 요소

역할:
- 어떤 파일을 읽어야 하는지 먼저 정리한다.
- 구현 전 필요한 문맥을 확보한다.

## Coder

입력:
- PlannerOutput
- 조사된 관련 파일 정보

출력:
- 변경 파일 목록
- 테스트 명령
- 진행 상태(`complete`, `partial`, `blocked`)

역할:
- 실제 코드 수정 계획을 구체화한다.
- 변경 범위와 테스트 명령을 정리한다.

## QA

입력:
- Coder 결과
- 테스트 명령

출력:
- 테스트 결과 요약
- 실패 여부
- 재시도 필요 여부

역할:
- 구현 결과가 요구사항을 만족하는지 확인한다.
- 실패 시 어떤 단계로 되돌릴지 판단한다.

## Reviewer

입력:
- Planner 결과
- Coder 결과
- QA 결과

출력:
- 최종 승인 또는 보완 요청
- 문서화 코멘트

역할:
- 전체 흐름이 목적과 일치하는지 확인한다.
- 산출물이 제출 가능한 수준인지 점검한다.

## 멀티에이전트 장점

- 역할 분리가 명확해져 각 단계의 책임이 분명하다.
- Planner가 큰 목표를 구조화하고, Coder가 구현에 집중할 수 있다.
- QA와 Reviewer 단계를 추가하면 품질 검증 흐름이 자연스럽게 붙는다.
- 실패 원인을 단계별로 나눠 추적하기 쉬워진다.
