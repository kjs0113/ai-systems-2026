# 반복 오류 분석 보고서

## 개요
본 보고서는 Baseline 프롬프트로 수집한 30개의 출력 결과를 분석하여, 반복적으로 나타나는 오류 패턴을 3-tier 구조로 분류하였습니다.

**분석 기간**: 2026-04-19  
**테스트 케이스 수**: 30개  
**오류 발생 건수**: 27건 (90%)

---

## Tier 1: 오류 유형 (대분류)

### 1.1 이해 오류 (Misunderstanding)
- **정의**: 사용자가 제공한 조건이나 컨텍스트를 정확히 이해하지 못하고 일반적인 답변만 제공
- **발생 빈도**: 6건 (20%)

### 1.2 생성 오류 (Generation Error)
- **정의**: 답변 내용이 불완전하거나 구체성이 부족하며, 때로는 사실과 다른 정보 생성
- **발생 빈도**: 17건 (56.7%)

### 1.3 형식 오류 (Formatting Error)
- **정의**: 사용자가 요구한 출력 형식(JSON, 리스트, 단계별 등)을 준수하지 않음
- **발생 빈도**: 4건 (13.3%)

---

## Tier 2: 세부 유형

### 2.1 이해 오류 → 조건 누락

**문제**: 사용자가 명시한 특정 조건(주문번호, 수량, 지역, 기간 등)을 무시하고 일반적인 답변만 제공

**심각도**: ⭐⭐⭐ (높음)

**발생 빈도**: 6건

**영향**:
- 사용자가 제공한 정보를 활용하지 못함
- 맞춤형 답변 불가능
- 사용자 불만족 증가

---

### 2.2 생성 오류 → 불완전 답변

**문제**: 답변에 필요한 핵심 정보가 누락되거나 구체적인 수치/절차가 생략됨

**심각도**: ⭐⭐⭐ (높음)

**발생 빈도**: 15건

**영향**:
- 사용자가 추가 질문을 해야 함
- 한 번에 문제 해결 불가
- 서비스 품질 저하

---

### 2.3 생성 오류 → 사실 왜곡 (Hallucination)

**문제**: 확인되지 않은 정보를 마치 사실처럼 제공

**심각도**: ⭐⭐⭐⭐ (매우 높음)

**발생 빈도**: 2건

**영향**:
- 신뢰도 심각한 손상
- 잘못된 기대 형성
- 고객 불만 폭발 위험

---

### 2.4 형식 오류 → 포맷 미준수

**문제**: JSON, 리스트, 단계별 등 명시된 출력 형식을 무시하고 일반 문장으로 답변

**심각도**: ⭐⭐ (중간)

**발생 빈도**: 4건

**영향**:
- 자동화 파이프라인 파손
- 파싱 오류 발생
- 후속 처리 불가

## 핵심 발견 사항

### 1. 조건 반영 능력 부재
- 사용자가 제공한 **주문번호, 회원등급, 지역, 수량** 등을 답변에 반영하지 못함
- 일반적인 답변만 생성하는 경향

### 2. 구체성 결여
- **"일정 금액", "특정 조건", "제품에 따라 다름"** 같은 모호한 표현 남발
- 사용자가 원하는 구체적 숫자/시간/절차 누락

### 3. 불확실성 처리 실패
- 모르는 정보를 솔직히 인정하지 않고 **추측**하여 답변
- 신뢰도 심각한 손상

### 4. 형식 인식 부족
- JSON, 리스트, 단계별 등 명시적 요구에도 일반 텍스트로 답변
- 자동화 시스템과의 연동 불가

---

## 개선 방향

### 1. Always Do 규칙
- ✅ 사용자가 제공한 조건(번호, 등급, 지역 등)을 반드시 답변에 명시
- ✅ 구체적 숫자/시간/금액이 필요하면 명확히 제시
- ✅ 정보를 모를 경우 "확인할 수 없습니다"라고 명시

### 2. Never Do 규칙
- ❌ 조건을 무시하고 일반적인 답변만 제공하지 마라
- ❌ "일정 금액", "제품에 따라" 같은 모호한 표현을 쓰지 마라
- ❌ 확인되지 않은 정보를 추측하여 답변하지 마라
- ❌ 요구된 출력 형식(JSON, 리스트, 단계)을 무시하지 마라

### 3. 출력 형식 준수
- JSON 요청 → `{"key": "value"}` 형태
- 리스트 요청 → `- 항목1`, `- 항목2`
- 단계별 요청 → `1단계:`, `2단계:`

---

## 결론

Baseline 분석 결과, **90%의 답변에서 오류**가 발견되었으며, 특히 **조건 누락(20%)**, **불완전 답변(50%)**, **포맷 미준수(13.3%)**가 반복적으로 나타났습니다.

이러한 패턴은 명확한 인스트럭션 튜닝을 통해 개선 가능하며, **Always Do / Never Do 규칙**을 적용하면 정확도, 완전성, 형식 준수율을 크게 향상시킬 수 있을 것으로 예상됩니다.

다음 단계에서는 이 분석 결과를 바탕으로 **PROMPT.md**를 설계하고, A/B 테스트를 통해 개선 효과를 정량적으로 측정하겠습니다.
