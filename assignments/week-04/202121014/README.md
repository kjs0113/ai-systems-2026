# Lab 04 : Ralph 루프 구현 

## 1. 개요

본 과제는 버그가 있는 `src/calculator.py`를 AI 에이전트 기반 Ralph Loop를 통해 자동으로 수정하고, 반복적인 실패와 재시도를 통해 문제를 해결하는 과정을 구현하는 것을 목표로 한다.

---

## 2. 목표

- 테스트 코드를 수정하지 않고 버그를 해결한다.
- 모든 pytest 테스트를 통과한다.
-  실패 → 학습 → 재시도 → 성공의 반복 구조를 구현한다.
-  반복 실행을 통해 성능이 향상되는 과정을 관찰한다.

---

## 3. 하네스 설계 (Harness Design)

본 과제에서는 `harness.sh`를 중심으로 자동 반복 실행 구조를 설계하였다.

### 3.1 반복 루프 구조

- 최대 반복 횟수(`MAX_ITER`) 설정
- 각 반복(iteration)마다:

  1. 테스트 실행
  2. 실패 시 Codex 호출
  3. 수정 후 재테스트
  4. 성공 시 종료

### 3.2 실패 유도 전략

반복 학습 구조를 명확히 보여주기 위해 의도적으로 실패 상태를 주입하였다.

- Iteration 1:

  - `ZeroDivisionError` 발생 (예외 처리 없음)
- Iteration 2:

  - 잘못된 예외 (`Exception("error")`)
- Iteration 3:

  - 올바른 `ValueError("0으로 나눌 수 없습니다")`로 수정

이를 통해 최소 2회 실패 후 성공하는 구조를 구현하였다.

### 3.3 Backpressure 및 제어

- 반복 간 `sleep`을 통해 과도한 호출 방지
- 로그 기록(`harness.log`)을 통해 각 단계 추적
- `DONE.md`를 통해 종료 조건 관리

## 4. Test-Time Compute Scaling과의 연결

본 실습은 **Test-Time Compute Scaling** 개념과 직접적으로 연결된다.

### 4.1 개념

Test-Time Compute Scaling은 모델을 한 번만 실행하는 것이 아니라,
여러 번 반복 실행하여 더 나은 결과를 얻는 방식이다.

### 4.2 본 과제에서의 적용

본 과제에서는 다음과 같은 방식으로 적용되었다:

- 단일 실행:

  - 한 번의 추론으로 해결 (실패 가능)
- 반복 실행 (Ralph Loop):

  - 여러 번 실행하며 점진적으로 수정
  - 실패를 통해 문제를 학습
  - 점점 정답에 가까워짐

### 4.3 효과

- 초기 실패를 통해 문제를 탐색
- 잘못된 해결 시도 수정
- 최종적으로 정확한 해결 도달

즉, **추론 횟수를 늘려 성능을 향상시키는 구조**를 확인할 수 있었다.
