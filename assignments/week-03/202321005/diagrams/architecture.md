# MCP 신뢰 경계 + TBAC 3계층 아키텍처 다이어그램

## 1. 전체 아키텍처: 3-Tier Trust Boundary

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        TRUST BOUNDARY 1                                │
│                   User ↔ Host ↔ Client                                 │
│  ┌──────────┐    ┌──────────────────┐    ┌──────────────────┐          │
│  │          │    │                  │    │                  │          │
│  │   User   │───▶│   Host (LLM)    │───▶│   MCP Client     │          │
│  │ (Human)  │    │  Claude / GPT   │    │  (SDK Runtime)   │          │
│  │          │◀───│                  │◀───│                  │          │
│  └──────────┘    └──────────────────┘    └────────┬─────────┘          │
│                                                    │                   │
│  · User는 Host를 신뢰함                            │                   │
│  · Host는 User의 의도를 해석                        │                   │
│  · Client는 Host의 도구 호출 결정을 실행             │                   │
│  · 이 경계 내에서 도구 승인/거부 결정                 │                   │
└────────────────────────────────────────────────────┼───────────────────┘
                                                     │
                              JSON-RPC (stdio/SSE)   │
                                                     ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                        TRUST BOUNDARY 2                                │
│                Client ↔ Gateway ↔ Server                               │
│                                                                        │
│  ┌──────────────┐   ┌─────────────────────┐   ┌──────────────────┐    │
│  │              │   │  Governed MCP        │   │                  │    │
│  │  MCP Client  │──▶│  Gateway (FastAPI)   │──▶│   MCP Server     │    │
│  │              │   │                      │   │  (FastMCP)       │    │
│  │              │◀──│  · 인바운드 검열      │◀──│                  │    │
│  └──────────────┘   │  · TBAC 정책 적용    │   │  · Tools         │    │
│                     │  · Rate Limiting     │   │  · Resources     │    │
│                     │  · 인젝션 탐지        │   │  · Prompts       │    │
│                     │  · 아웃바운드 검열     │   │                  │    │
│                     │  · 감사 로그          │   └──────────────────┘    │
│                     └─────────────────────┘                            │
│                                                                        │
│  · Client는 Server를 직접 신뢰하지 않음                                 │
│  · Gateway가 TBAC 정책을 기반으로 접근 제어                              │
│  · 도구 설명(description)은 untrusted 데이터로 취급                      │
│  · 프롬프트 인젝션 패턴을 Gateway에서 탐지/차단                          │
└────────────────────────────────────────────────────┬───────────────────┘
                                                     │
                              pynvml / API calls      │
                                                     ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                        TRUST BOUNDARY 3                                │
│                   Server ↔ Infrastructure                              │
│                                                                        │
│  ┌──────────────────┐   ┌──────────────────────────────────┐          │
│  │                  │   │        DGX H100 Infrastructure    │          │
│  │   MCP Server     │──▶│                                  │          │
│  │   (FastMCP)      │   │  ┌────────────────────────────┐  │          │
│  │                  │   │  │ GPU 0 (MIG Enabled)        │  │          │
│  │  · pynvml API    │   │  │  ┌──────┐ ┌──────┐        │  │          │
│  │  · 입력 검증     │   │  │  │1g.10g│ │1g.10g│ ...×7  │  │          │
│  │  · 에러 핸들링   │   │  │  └──────┘ └──────┘        │  │          │
│  │                  │◀──│  └────────────────────────────┘  │          │
│  └──────────────────┘   │                                  │          │
│                         │  · nvidia-smi mig -lgi           │          │
│                         │  · NVML C Library                │          │
│                         │  · Kubernetes Device Plugin      │          │
│                         └──────────────────────────────────┘          │
│                                                                        │
│  · Server는 인프라에 대해 최소 권한(Least Privilege) 적용               │
│  · GPU 인덱스 범위 검증으로 잘못된 하드웨어 접근 방지                    │
│  · pynvml 초기화 실패 시 안전한 fallback (mock 데이터)                  │
└─────────────────────────────────────────────────────────────────────────┘
```

## 2. TBAC (Task-Based Access Control) 흐름: Tasks → Tools → Transactions

```
┌─────────────────────────────────────────────────────────────────┐
│                    TBAC 3-Layer Flow                            │
│                                                                 │
│  ┌───────────────────────────────────────────────────┐         │
│  │ Layer 1: TASKS (태스크 정의)                        │         │
│  │                                                     │         │
│  │  ┌─────────────┐  ┌─────────────┐  ┌────────────┐ │         │
│  │  │ monitoring  │  │  analysis   │  │ admin      │ │         │
│  │  │ (조회)      │  │  (분석)     │  │ (관리)     │ │         │
│  │  └──────┬──────┘  └──────┬──────┘  └─────┬──────┘ │         │
│  │         │                │                │        │         │
│  └─────────┼────────────────┼────────────────┼────────┘         │
│            │                │                │                  │
│            ▼                ▼                ▼                  │
│  ┌───────────────────────────────────────────────────┐         │
│  │ Layer 2: TOOLS (도구 매핑)                          │         │
│  │                                                     │         │
│  │  monitoring ──▶ get_mig_status                     │         │
│  │               ▶ get_gpu_metrics                    │         │
│  │                                                     │         │
│  │  analysis ────▶ analyze_gpu_usage                  │         │
│  │               ▶ gpu_analysis_prompt                │         │
│  │                                                     │         │
│  │  admin ───────▶ admin_reset_gpu                    │         │
│  │               ▶ admin_reconfigure_mig              │         │
│  └──────────────────────────┬────────────────────────┘         │
│                              │                                  │
│                              ▼                                  │
│  ┌───────────────────────────────────────────────────┐         │
│  │ Layer 3: TRANSACTIONS (실행 + 감사)                 │         │
│  │                                                     │         │
│  │  Tool Call ──▶ Gateway 검증 ──▶ TBAC 권한 확인     │         │
│  │           ──▶ 입력 검증 ──▶ 실행 ──▶ 감사 로그     │         │
│  │                                                     │         │
│  │  ┌─────────────────────────────────────────┐       │         │
│  │  │ Transaction Record:                      │       │         │
│  │  │  timestamp: 2026-03-20T14:30:00+09:00   │       │         │
│  │  │  role: student                           │       │         │
│  │  │  tool: get_mig_status                    │       │         │
│  │  │  task: monitoring                        │       │         │
│  │  │  result: allowed                         │       │         │
│  │  │  params: {gpu_index: 0}                  │       │         │
│  │  └─────────────────────────────────────────┘       │         │
│  └───────────────────────────────────────────────────┘         │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 역할별 권한 매트릭스

```
              ┌──────────────┬──────────────┬──────────────┐
              │   student    │  researcher  │    admin     │
┌─────────────┼──────────────┼──────────────┼──────────────┤
│ monitoring  │      ✅      │      ✅      │      ✅      │
│ (조회)      │              │              │              │
├─────────────┼──────────────┼──────────────┼──────────────┤
│ analysis    │      ✅      │      ✅      │      ✅      │
│ (분석)      │              │              │              │
├─────────────┼──────────────┼──────────────┼──────────────┤
│ admin       │      ❌      │      ❌      │      ✅      │
│ (관리)      │  DENIED      │  DENIED      │              │
└─────────────┴──────────────┴──────────────┴──────────────┘
```
